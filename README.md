# Invoice Management System (InvoAI)

This project is a course assignment deliverable for an invoice management platform with end-to-end support for upload, OCR, storage, search, analytics, and AI assistance.

## Core Features

- Upload invoice files (image/PDF)
- Extract and store invoice fields via OCR
- Search invoices by date range and fuzzy keyword match
- Show statistics (invoice count and total amount)
- Click an invoice to preview the original image/PDF

## Tech Stack

- Client: Vite + React + TypeScript + Tailwind CSS
- Desktop app (optional): Tauri
- Server API: Next.js Route Handlers
- Database: MySQL 5.6+

## Repository Structure

- `src/`: frontend client
- `server/`: backend API and database access
- `src-tauri/`: Tauri desktop wrapper and local file commands
- `scripts/qa-check.mjs`: automated QA script (supports long-running cycles)
- `reports/`: CSV validation outputs

## Environment Variables

### Frontend `.env`

```env
VITE_API_BASE=http://localhost:3000
```

### Backend `server/.env.local`

```env
DATABASE_URL=mysql://root:root@localhost:4000/invoice_db
JWT_SECRET=your-jwt-secret
AI_API_KEY=your-ai-key
AI_API_BASE=https://api.openai-next.com
```

## Run Locally

### 1) Start backend

```bash
cd server
npm install
npm run dev
```

### 2) Initialize database (first run)

```bash
curl -X POST http://localhost:3000/api/init
```

### 3) Start frontend

```bash
cd ..
npm install
npm run dev
```

## Validation and QA

### Single-run validation

```bash
npm run qa
```

This runs:

- `npm run lint`
- frontend `npm run build`
- server `npm run build`

Results are appended to: `reports/qa-runs.csv`

### Long-running soak validation

```bash
npm run qa:soak
```

Default: 12 cycles with 300-second interval.

Custom example:

```bash
node scripts/qa-check.mjs --repeat=24 --interval=120 --report=reports/qa-nightly.csv
```

## GitHub Release Packaging

Cross-platform desktop installers are generated by GitHub Actions:

- Workflow: `.github/workflows/release-desktop.yml`
- Trigger: push a tag that starts with `v` (for example `v1.0.1`)
- Output: release assets for macOS, Windows, and Linux

Example:

```bash
git tag v1.0.1
git push origin v1.0.1
```
